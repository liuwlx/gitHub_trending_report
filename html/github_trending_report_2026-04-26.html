⭐9,547 🍴1,201 CUDA / C++ 📄 MIT

📖 项目摘要

DeepEP 是一个为 Mixture-of-Experts（MoE）模型设计的通信库，专门优化专家并行（Expert Parallelism）场景下的 all-to-all 通信（MoE dispatch 和 combine）。提供两套内核：NVLink + RDMA 异域转发高吞吐内核（训练和推理 prefilling），以及纯 RDMA 低延迟内核（推理 decoding）。同时提供不占用 SM 资源的 hook-based 通信-计算重叠方案。

✨ 核心特性

  • 双路径内核：NVLink 域到 RDMA 域的高吞吐异域转发（训练/prefilling）+ 纯 RDMA 低延迟内核（decoding），覆盖两种主要 MoE 运行场景
  • 低精度支持：原生支持 FP8，对齐 DeepSeek-V3 的 group-limited gating 算法，同时支持 SM 数量控制
  • 通信-计算重叠：hook-based 重叠方案不占用任何 SM 资源，在计算过程中完成通信，减少等待时间

📖 项目摘要

Universal Commerce Protocol（UCP）是一个开放标准，旨在为不同的商业实体（电商平台、AI Agent 应用、支付服务提供商、凭据提供商）之间提供统一的通信协议，解决当前 AI 代购场景下各平台无法互通的问题。UCP 把商业能力模块化（Checkout、Order 等），支持 AI Agent 自主发现商家能力、发起结账会话，并支持 AP2 授权命令和可验证凭据等高级安全模式。

✨ 核心特性

  • AI 代理友好设计：从底层设计上支持 AI Agent 自主发现商家能力并完成购物，而不需要每个商家单独为 AI 定制接入
  • 模块化商业能力：把 Checkout、Order、Discounts、Fulfillment 等分拆为独立 Capability 和 Extension，商家按实际支持范围实现
  • 安全机制内嵌：支持 AP2 授权命令（mandate）和可验证凭据，为无人值守的 AI 代购场景提供安全保障
